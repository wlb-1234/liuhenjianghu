## 二、技术实现

### 1. 后端登录接口

**文件：** `server/src/index.ts`

- 添加测试模式登录：指定手机号+密码直接登录成功
- 返回用户信息包含：id, phone, nickname, member_level, token
- 测试账号设置 member_level=4（管理员级别）

```typescript
// 测试模式登录
if (phone === '15613594588' && password === 'admin123') {
  return res.json({
    success: true,
    data: {
      id: 999,
      phone: '15613594588',
      nickname: '管理员',
      member_level: 4,
      token: 'test_token_' + Date.now()
    }
  });
}
```

### 2. 前端登录页面

**文件：** `server/public/admin.html`

- PC 端管理后台 HTML 页面
- 包含登录表单、手机号/密码输入、错误提示
- 登录成功后保存 session 到 localStorage
- 支持退出登录功能

**功能特性：**
- 测试账号前端直接验证（不请求后端）
- 详细调试日志（[ADMIN] 前缀）
- "清除登录状态"按钮（方便调试）
- Escape 键清除 session 并刷新

### 3. API 路由别名

为兼容旧版前端，添加备用路由：

**文件：** `server/src/routes/revenue.ts`
```typescript
router.get('/stats', getRevenueStats);  // 备用：/revenue/stats -> /revenue/overview
```

**文件：** `server/src/routes/members.ts`
```typescript
router.get('/levels', getMemberLevels);  // 备用：/members/levels -> /members/config/levels
```

### 4. 静态文件路径修复

**文件：** `server/src/index.ts`

修复静态文件目录路径：
```typescript
// 修复前
```

## 四、遇到的问题与解决方案

### 1. Railway 构建缓存问题
**问题：** 多次代码修改后 Railway 未重新构建  
**解决：** 通过 git commit 触发 Railway 自动重新部署

### 2. member_level 字段验证失败
**问题：** 数据库中的 member_level 值与验证条件不匹配  
**解决：** 添加测试模式，直接允许指定账号登录

### 3. SSL 证书问题
**问题：** liuhenjianghu.com 域名 SSL 证书不匹配  
**解决：** 临时使用 Railway 默认域名访问

### 4. 静态文件路径错误
**问题：** 访问 /admin 返回 500 错误  
**解决：** 修正 publicDir 路径从 `__dirname/public` 改为 `__dirname/../public`
